# Future Ideas & Architectural Improvements

Этот документ содержит идеи по улучшению проекта **Claude/Gemini/Codex Orchestrator**, вдохновленные лучшими практиками изученных репозиториев (orchestrate, skill-codex, bridge).

---

## 🏗 Основные архитектурные концепции

### 1. Decision Engine (Интеллектуальный роутинг)
*   **Идея:** Автоматический "измеритель сложности" для выбора модели.
*   **Реализация:** Скрипт `Measure-TaskComplexity.ps1`, который сканирует объем файлов и промпта.
*   **Профит:** Экономия дорогого контекста Claude; перенаправление тяжелых задач в Gemini 1.5/2.0 Pro.

### 2. Dynamic Skill Discovery (YAML в Markdown)
*   **Идея:** Использование YAML frontmatter в профилях агентов для описания "навыков".
*   **Реализация:** Расширение файлов в `roles/*.md` полями `capabilities`, `max_context` и `tools_allowed`.
*   **Профит:** Масштабируемость без правки кода — достаточно добавить новый .md файл роли.

### 3. Очистка контекста (Sanitization)
*   **Идея:** Удаление токенов размышлений (`<thinking>`) и лишних логов при передаче "эстафеты" между моделями.
*   **Реализация:** Фильтр-обертка в `Invoke-Flow.ps1`.
*   **Профит:** Чистый контекст и экономия токенов при передаче данных от Claude к Gemini/Codex.

### 4. Умное кэширование (Hashing)
*   **Идея:** Хеширование файлов для предотвращения повторной обработки неизмененного кода.
*   **Реализация:** SHA-256 хеши в директории `.orchestrate/cache/`.
*   **Профит:** Ускорение итераций и снижение затрат на API.

## ⚡ Рабочие процессы и устойчивость

### 5. Session Persistence & Resume (Чекпоинты)
*   **Идея:** Сохранение полного состояния выполнения в `session.json`.
*   **Реализация:** Флаг `--Resume <SessionID>` для продолжения прерванной задачи.
*   **Профит:** Устойчивость к сбоям сети и возможность "дожать" задачу на следующий день.

### 6. Ensemble Verification (Перекрестная проверка)
*   **Идея:** Параллельный запуск Claude и Gemini для критических участков кода.
*   **Реализация:** Скрипт-компаратор, ищущий расхождения в архитектурных решениях.
*   **Профит:** Сведение "галлюцинаций" к минимуму.

### 7. Project Atlas (Глобальная карта проекта)
*   **Идея:** Периодическая генерация Gemini высокоуровневой карты всего репозитория.
*   **Реализация:** Подмешивание "атласа" в системный промпт легким моделям (Haiku).
*   **Профит:** Легкие агенты понимают контекст всего проекта, не читая сотни файлов.

## 🛡 Безопасность и автономия

### 8. Self-Healing Loop (Авто-исправление)
*   **Идея:** Автоматическая доработка кода на основе ошибок CLI.
*   **Реализация:** Если тесты/линтер упали, логи ошибок летят обратно агенту на новую итерацию.
*   **Профит:** Полная автономия в исправлении тривиальных ошибок.

### 9. Role-Based Sandboxing (Ограничение инструментов)
*   **Идея:** Строгие ограничения на инструменты в зависимости от роли (например, Reviewer не может Edit).
*   **Реализация:** Блокировка доступа к инструментам на уровне PowerShell-обертки.
*   **Профит:** Безопасность — агент не сломает код, если его задача только анализ.
