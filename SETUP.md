# Claude Dispatcher v2: Setup & Architecture

Инструмент для оркестрации CLI-агентов (Claude, Gemini, Codex) в среде Windows 11.

## 1. Режимы работы

### Режим A: Global (Пользовательский)
Проект устанавливается один раз в вашу домашнюю директорию и доступен во всех репозиториях.
- **Путь:** `%USERPROFILE%\.claude\`
- **Как включить:** `powershell .\Install-Dispatcher.ps1 -Mode Global`
- **Плюсы:** Глобальные хуки, единые скиллы и агенты для всех ваших задач.

### Режим B: Local (Проектный)
Проект копируется/настраивается индивидуально для конкретного репозитория.
- **Путь:** `<your-repo>\.claude\`
- **Как включить:** `powershell .\Install-Dispatcher.ps1 -Mode Local`
- **Плюсы:** Изолированные настройки цепочек инструментов для конкретного проекта.

---

## 2. Четыре формата интеграции

### 1. Directives (claude.md)
Инструкции в файле `CLAUDE.md`, которые заставляют Claude Code работать как Оркестратор. Он планирует задачу и вызывает цепочку через `Invoke-Chain.ps1`.
- **Файл:** `CLAUDE.md` (в корне или глобально).

### 2. Hooks (Хуки)
Автоматический перехват событий Claude CLI (например, перед использованием инструментов или после ввода промпта).
- **Файл:** `%USERPROFILE%\.claude\settings.json` (только для Global режима).

### 3. Skills (Скиллы)
Добавление новых "умений" в Claude Code через JavaScript-функции, обернутые в PowerShell-скрипты.
- **Папка:** `.claude\skills\`

### 4. Sub-agents (Сабагенты)
Использование специализированных профилей агентов (`researcher`, `coder`, `reviewer`) с преднастроенными ролями и ограничениями.
- **Папка:** `.claude\agents\`

---

## 3. Настройка цепочек (flow.config.json)

Цепочки вызовов описываются в `flow.config.json`. Вы можете задать последовательность инструментов, модели и роли:

```json
{
  "flows": {
    "my_flow": {
      "steps": [
        { "tool": "gemini", "role": "researcher" },
        { "tool": "codex", "role": "implementer", "yolo": true }
      ]
    }
  }
}
```

---

## 4. Совместимость и Требования
- **OS:** Windows 10/11 (никакого WSL/Linux).
- **Shell:** PowerShell 5.1 (Native) или PowerShell 7.1+ (Core).
- **Инструменты:** Официальные `claude`, `gemini`, `codex` CLI должны быть установлены и доступны в PATH.

---

## 5. Расширяемость
Чтобы добавить новый инструмент (например, `OpenCode`):
1. Добавьте его в `tools` в `flow.config.json`.
2. Опишите шаг в `flows`.
3. Система автоматически подхватит его при вызове `Invoke-Chain`.
