# Claude Dispatcher v2: Setup & Architecture

Инструмент для оркестрации CLI-агентов (Claude Code, Gemini CLI, Codex CLI) в среде Windows 11. 

---

## 1. Режимы работы (Scope)

Режим определяет, **где хранятся настройки** и **на какие проекты** они влияют.

### Режим: Global (Глобальный)
Проект устанавливается в вашу домашнюю директорию и работает как «надстройка» над Claude Code для всех папок на компьютере.
- **Путь:** `%USERPROFILE%\.claude\`
- **Файлы:** Настройки, роли и хуки применяются ко всем вызовам `claude`.
- **Применение:** Идеально для работы в любых папках без необходимости каждый раз настраивать проект.

### Режим: Local (Проектный)
Настройки изолированы внутри конкретного Git-репозитория.
- **Путь:** `.claude\` в корне вашего проекта.
- **Файлы:** Позволяет создавать уникальные цепочки инструментов (Flows) и специфические роли для конкретных задач проекта.

---

## 2. Типы интеграции (Formats)

Тип определяет, **как именно** Dispatcher встраивается в рабочий процесс Claude Code.

1.  **Directives (`Directives`)**: Инструкции в файле `CLAUDE.md`. Claude Code всегда видит их как контекст и следует правилам делегирования.
2.  **Hooks (`Hooks`)**: Автоматический перехват ввода пользователя через `UserPromptSubmit`. Работает **только в Global режиме** (через `~/.claude/settings.json`). Инъекция инструкций происходит динамически в каждый промпт.
3.  **Skills (`Skills`)**: Добавление новых JavaScript-функций в Claude Code. Позволяет вызывать `Invoke-Flow.ps1` как инструмент (tool).
4.  **Agents (`Agents`)**: Использование набора системных промптов (roles) в `~/.claude/agents/`, которые превращают Claude в "Strategic Planner".

---

## 3. Планирование в файл (Planning to File)

В версии v2 внедрен обязательный этап планирования для повышения надежности выполнения сложных задач.

### Как это работает:
1.  **Конфигурация**: В `project.settings.json` должна быть включена секция:
    ```json
    "planning": {
      "enabled": true,
      "dirName": "plans"
    }
    ```
2.  **Инструкция**: Claude получает указание: «Прежде чем что-то менять, создай файл плана».
3.  **Действие**: Claude использует `write_file`, чтобы сохранить план в директорию `plans/` (например, `plans/plan_fix_auth_20240308.md`).
4.  **Выполнение**: Сразу после сохранения плана Claude вызывает `Invoke-Flow.ps1`, передавая только путь к этому файлу.

### Преимущества:
- **Контроль**: Вы всегда видите, что именно собирается сделать агент.
- **История**: Все планы сохраняются в проекте.
- **Точность**: Внешние агенты (Gemini/Codex) читают файл плана и следуют ему буквально.

## 4. Управление настройками (`project.settings.json`)

Файл `project.settings.json` — это пульт управления Dispatcher. 

### Основные поля:
- `"mode"`: `"global"` или `"local"`.
- `"activeFormat"`: Текущий активный формат (directives, hooks, skills, subagents).
- `"formats"`: Объект включения/выключения конкретных механизмов.
- `"planning"`: Настройки режима планирования.

### Как переключать режимы:

Используйте скрипт `scripts\Switch-Mode.ps1`. Он работает по принципу **XOR** — включает один выбранный формат и выключает остальные, чтобы избежать конфликтов инструкций.

| Задача | Команда PowerShell |
| :--- | :--- |
| **Global + Hooks** (Рекомендуемый) | `.\scripts\Switch-Mode.ps1 -Mode Hooks -Scope Global` |
| **Local + Directives** | `.\scripts\Switch-Mode.ps1 -Mode Directives -Scope Local` |
| **Global + Agents** | `.\scripts\Switch-Mode.ps1 -Mode Agents -Scope Global` |

## 5. Настройка цепочек (`flow.config.json`)

Этот файл определяет, какие инструменты будут вызваны при выполнении задачи.

```json
{
  "defaultFlow": "standard",
  "flows": {
    "standard": {
      "steps": [
        { "name": "research", "tool": "gemini", "role": "researcher" },
        { "name": "implement", "tool": "codex", "role": "implementer", "yolo": true }
      ]
    }
  }
}
```

- **`tool`**: `gemini` (умный), `codex` (быстрый), `claude` (сбалансированный).
- **`role`**: Системный промпт из папки `roles/`.
- **`yolo`**: `true` отключает подтверждения для этого шага.

---

## 6. Полезные скрипты

1.  **`Invoke-Flow.ps1`** — Главный исполнитель. Автоматически распознает, передана ли ему строка задачи или путь к файлу плана.
2.  **`Switch-Mode.ps1`** — Переключатель "все в одном". Обновляет JSON, копирует файлы в профиль и чистит `CLAUDE.md`.
3.  **`measure_speed.py`** — Скрипт для тестирования скорости ответа различных LLM в вашей среде.

---

## 7. Совместимость
- **OS:** Windows 10/11 (Native PowerShell 5.1 / 7.x).
- **Инструменты:** CLI-версии `claude`, `gemini`, `codex` должны быть установлены и доступны в PATH.
